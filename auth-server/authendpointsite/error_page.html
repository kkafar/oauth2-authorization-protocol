Authorization endpoint errors

Invalid content type

Source: HttpHeadersValidator

Thrown when Content-Type doesn't match text/html nor application/x-www-form-urlencoded.

Repeating get parameters error

Source: RepeatingGetParametersChecker

Thrown when get parameter is repeated.

Missing client_id parameter

Source: RedirectionUriVerifier

Thrown when client_id parameter isn't present in URI.

Missing redirect_uri parameter

Source: RedirectionUriVerifier

Thrown when redirect_uri parameter isn't present in URI.

Unknown client_id

Source: RedirectionUriVerifier

Thrown when client_id is unknown to server database.

client_id redirect_uri mismatch

Source: RedirectionUriVerifier

Thrown when given redirect_uri doesn't match redirect_uri of client with given client_id.

Missing state parameter

Source: StateValidator

Thrown when state parameter isn't present in URI.

Malformed state parameter

Source: StateValidator

Thrown when state parameter doesn't match [ -~]+ regex.

Missing response_type parameter

Source: ResponseTypeVerifier

Thrown when response_type parameter is missing.

Unknown response type

Source: ResponseTypeVerifier

Thrown when response type doesn't match "code".

Missing scope parameter

Source: ScopeValidator

Thrown when scope parameter is missing.

Unknown scope

Source: ScopeValidator

Thrown when one or more of space separated scope entries are unknown.

Missing code_challenge parameter

Source: CodeChallengeValidator

Thrown when code_challenge parameter is missing.

Invalid code challenge parameter

Source: CodeChallengeValidator

Thrown when code_challenge doesn't match this regex [a-zA-Z0-9_\-.~]+.

Unknown code challenge method

Source: CodeChallengeValidator

Thrown when code_challenge_method is present and doesn't match "plain" nor "s256".
